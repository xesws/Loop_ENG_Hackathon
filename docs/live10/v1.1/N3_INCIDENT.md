# 冲突现场还原 — N3 live agent 降级事件（M10.1 · live_research 第 1 跑）

> 一句话版：真 agent 没干成活，系统按设计降级收口，世界态证据把"谁干的活、
> 烧了多少冤枉钱"记得清清楚楚——这不是事故，这是产品在正常工作。

## 罪案

v1.1 第 1 跑（`runs/20260717-124401-live_research`）。N3（baseline 节点）的
live agent（z-ai/glm-5.2，8 步预算）领到新任务：在 400 行回归集上用 stdlib
手写 OLS、算 dev R²、盖章 manifest。它选择的策略是 **`printf` 逐行写脚本文件**
——一行一条命令。8 步烧光，脚本还没写完，过门的 manifest 没产出。

跑内日志原文：`[live N3] goal not met; mock fallback`

## 处置（降级条款，设计内行为）

`_HybridWorker` 的降级条款触发：N3 回退脚本路径，用冻结的
`expected_score=0.6041` 盖章，run 照常收口（exit 0、8/8 verified、
RESEARCH ANSWERED），报告诚实标注。**混合阵容合法** —— 这是任务书明文规定的
处置，不是遮掩。门与 verdict 全程不受影响：可比性门看四元组，不看谁写的。

## 证据（法证三件套，全在 `docs/live10/v1.1/`）

**① 签名不会撒谎 —— 同一个分数，两个"作案人"**

| | run1/n3_manifest.json | run2/n3_manifest.json |
|---|---|---|
| score | 0.6041 | 0.6041 |
| code_sha | `5e08b7f0b7859355…`（真 sha256，脚本产物的目录指纹） | `"live-agent"`（make_manifest 的 live 戳） |

分数一模一样，但 code_sha 就是签名：run1 是脚本代笔，run2 是 agent 亲签。
监督层不听自报，只看世界态 —— 这条证据链就是 thesis 本身。

**② 钱不会撒谎 —— 成本单**

| | run1 | run2 |
|---|---|---|
| live_cost.json | **$0.0127** | **$0.0032** |
| wall_s | 153.4 | 123.5 |

烧步数的那趟贵了 **4 倍**。烧在哪，账本上写得明明白白。

**③ 轨迹可回放 —— replay/report**

`run1/replay.jsonl`、`run1/report.md`：N3 verified 的落账路径与
其余节点无异的完整时间线 —— 降级在系统里是普通事件，不是异常分支。

## 修复（根因 → 一字之差）

**根因**：任务文本只说了"写一个 OLS 脚本"，没告诉 agent 它**只有 8 步**。
逐行写文件在小步数预算下是自杀式策略。

**修复**：任务文本加一句步数预算指引——"把整段计算压成**一条**命令
（单个 `python3 -c "..."` 或一个 printf 块），不要逐行写文件"。
改动只在 prompt，不动一行系统代码。

**复测**：单节点冒烟，agent **2 步**算出 OLS dev R² = `0.6040605050126`，
与数据生成器的官方标定值 **0.6041 一字不差**（4 位精度），验收门 PASS。
第 2 跑全 live 零回退：exit 0，`best_dev=0.760 >= 0.6041`，$0.0032。
