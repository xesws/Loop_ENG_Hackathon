# 联合分析 — 同一个 agent，两种表现；同一套系统，一种结果（run1 × run2）

> 一句话版：同一个 GLM-5.2、同一个任务、同一份预算，两跑走出了两条完全不同的
> 执行路径——一条翻车、一条满分。系统的行为两跑**完全一致**：观察、过门、
> 降级、落账、收口。agent 是不可靠的随机变量，监督层是常数。这就是产品的价值。

（现场记录：`N3_INCIDENT.md` = run1 翻车现场；`RUN2_FULL_LIVE.md` = run2
满分现场。本文是两者的联合分析。证据原件全在 `docs/live10/v1.1/`。）

## 实验设置（一个意外完美的 A/B）

- **同一模型**：z-ai/glm-5.2（OpenRouter）
- **同一任务**：N3 baseline 节点——400 行回归集上自写 OLS、算 dev R²、盖章 manifest
- **同一预算**：max_steps=8、API ≤$3、同一场景 live_research、同一份系统代码
- **唯一 delta**：run1 与 run2 之间，任务文本加了**一句**步数预算指引
  （"整段计算压成一条命令"）。系统代码零改动。
- LLM 采样的不确定性 + 这一句话，造就了两条路径。如实说明：run1 的翻车
  发生在加这句话**之前**——系统抓到问题时，手里没有任何"修复"。

## 对照表

| | run1（翻车） | run2（满分） |
|---|---|---|
| agent 策略 | `printf` 逐行写脚本（自杀式） | 单条 `python3 -c` 一次算完 |
| 步数消耗 | 8/8 烧光，manifest 未产出 | 2/8，OLS R²=0.6040605050126 |
| 系统处置 | 降级条款：脚本代笔（冻结值 0.6041） | 无需处置：agent 亲签 |
| N3 code_sha | `5e08b7f0b7859355…`（脚本指纹） | `live-agent` |
| 成本 | $0.0127 | $0.0032（4 倍差） |
| 墙钟 | 153.4s | 123.5s |
| **exit / verdict** | **exit 0 · 8/8 verified · RESEARCH ANSWERED 0.760≥0.6041** | **完全一致** |

## 系统抓到的是什么（三层证明）

**① 发现（detection）**：agent 说没说"我完了"不重要——`goal not met` 是对
世界态的检查（manifest 存在且合法吗），不是对 agent 自报的采信。没产出就是
没产出，降级条款当场触发。

**② 遏制（containment）**：翻车被关进一个节点里。run1 照常 exit 0、
8/8 verified、结论落账——降级是五档阶梯里的设计档位，不是异常分支；
混合阵容是任务书明文的合法形态。故障没有扩散到 verdict。

**③ 可诊断（diagnosability）**：不看任何 transcript，仅凭落盘证据就能
断案——code_sha 说出"谁干的活"（脚本指纹 vs `live-agent` 亲签），成本单
说出"冤枉钱烧在哪"（4 倍差），replay.jsonl 说出"降级发生在哪一拍"。
根因（任务文本没给步数预算）因此是一眼的事，修复只需要一句话。

## 不变量（两跑完全一致的系统行为）

- exit 0、quiesced、8/8 verified、incidents 0
- verdict 数值一模一样：`best_dev=0.760 >= 0.6041`
- 可比性门两跑都拿真 manifest 验四元组，逐项相等才放行——不看谁写的
- 账本两跑都如实落盘（live_cost.json），包括翻车那趟的冤枉钱

## 变量（agent 的两次不同表现）

策略选择、步数消耗、成本、manifest 签名——全是 agent 侧的自由变量。
LLM 的本质就是不可复现；你无法修好模型本身，只能修好**接住它的那一层**。

## 30 秒讲者话术

"我们让同一个 GLM-5.2 在同一个任务上跑了两遍。第一遍它选了个自杀式策略，
八步烧光；系统没听它辩解，查了世界态，按设计降级，跑完全绿，证据落盘。
第二遍它两步就算出了和官方标定一字不差的 0.6041。请注意两跑的 verdict
一模一样——agent 是随机变量，监督层是常数。你买的不是 agent 不犯错，
你买的是**它犯什么错都逃不掉**。"
